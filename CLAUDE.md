# CLAUDE.md

## Project
Sightline — a cross-platform desktop app that aggregates GTA-RP VODs across streamers into one chronological, sync-playable timeline. **v1.0.0 shipped 2026-04-25**, **v2.0.0 shipped 2026-04-25** (storage-aware capstone), **v2.0.1 shipped 2026-04-26** (Phase 8 scope-closure: forecast UI, unified library, pre-fetch wiring, Windows suspend), **v2.0.2 shipped 2026-04-26** (sidecar-resolution hotfix per ADR-0034) — see `CHANGELOG.md` for release notes; `docs/STATE.md` for the live operating snapshot.

## Stack
- Rust 1.90 stable · Tauri 2 · Tokio · sqlx (SQLite) · reqwest
- React 19 · TypeScript 5 · Vite 6 · Tailwind CSS 4 · shadcn/ui
- TanStack Query · Zustand · tauri-specta (typed IPC)
- pnpm 9 · Vitest · Playwright (later phases)

## Commands
- `pnpm tauri dev` — run the app in development (hot-reload frontend + Rust rebuild)
- `pnpm tauri build` — produce signed platform installer
- `pnpm dev` — frontend only (no Rust shell)
- `pnpm typecheck` — `tsc --noEmit`
- `pnpm lint` — ESLint on `src/`
- `pnpm test` — Vitest
- `cargo fmt --check` · `cargo clippy --all-targets -- -D warnings` · `cargo test` — Rust quality gate
- `./scripts/bootstrap.sh` (or `.ps1`) — one-shot environment check + sidecar fetch

## Conventions
- **Commits**: Conventional Commits. One logical change per commit. Quality gate must pass on every commit.
- **Rust layering**: `commands` (thin, `#[tauri::command]`) → `services` (orchestration) → `domain` (pure types) → `infra` (DB, HTTP, sidecars). No back-edges.
- **Frontend layering**: feature folders under `src/features/`. Shared primitives in `src/components/`. Generated IPC bindings in `src/ipc/` are read-only.
- **Errors**: backend uses `thiserror` enums that implement `Serialize` for IPC. No `unwrap()` or `expect()` outside tests.
- **IPC contracts**: single source of truth in Rust; TS types generated by tauri-specta. Never hand-edit generated files.
- **Config**: application config lives in one `AppConfig` struct with per-OS default paths. Secrets go to the OS keyring, never to disk in plaintext.
- **Tests**: unit tests co-located with the code they test (`mod tests` in Rust, `.test.tsx` siblings in TS). Integration tests in `src-tauri/tests/`.

## When in doubt
1. Read the blueprint at `docs/reference/synthetic-workforce-blueprint.md`.
2. Check for a relevant ADR under `docs/adr/`.
3. If the decision isn't covered, propose a new ADR rather than deviating silently.
4. For destructive, irreversible, or shared-state operations: stop and surface to the human. The synthetic workforce is governed by the principle of reversibility by default.
