## Decision-log highlights

Top entries from `docs/decision-log/v2.0.1-scope-closure.md`:

1. **Windows suspend via PowerShell, not `windows-rs`.**  Keeps
   `unsafe_code = "forbid"` intact.  Trade-off: ~0.3-0.8 s startup
   cost per call, invisible at the 5 s throttle cadence.
2. **Pre-fetch throttle = app-session via module-level Set.**
   Promoted from `useRef` (per-component) after R-RC-01 caught the
   doc/implementation drift.  Survives HMR.
3. **Vods.status convergence via lib.rs sink + downloads helper,
   not service-to-service import.**  Keeps services decoupled at
   the cost of a 5 s tick latency before the worker observes new
   queued rows.
4. **`removeVod` is a new IPC, not a routing through cancelDownload.**
   ADR-0033 specified Remove as a new action; building the
   missing command was scope-disciplined per R-SC-01.
5. **Lifecycle filter chips operate client-side.**  ADR-0033
   doesn't mandate server vs client; client-side is faster on
   the 200-VOD limit and avoids re-querying on every event tick.

Plus a transparency note: a brief disk-pressure stop fired
mid-Sub-Phase D when `/tmp/claude-501` filled up.  Documented
separately in `docs/decision-log/v2.0.1-disk-stop.md`.  No code
state was lost; the working tree was preserved through the
recovery.
