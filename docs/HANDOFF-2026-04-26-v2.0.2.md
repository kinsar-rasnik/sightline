## Was wurde geliefert

**Trigger:** Real-User-Bug-Report von CEO am 2026-04-26 nach v2.0.1-Installation:
- Hardware-Encoder-Detection failed mit `sidecar: spawn: No such file or directory (os error 2)`
- Downloads stuck in RETRYING-State (alle 6 VODs)
- macOS "App is damaged"-Dialog beim ersten Start (xattr-Workaround nötig)
- Co-Streams-Panel hatte nur "Download & Watch" — kein "Download only"

**Diagnose:** `lib.rs::resolve_sidecar` suchte Sidecars unter `tauri::path::BaseDirectory::Resource` (= `Sightline.app/Contents/Resources/binaries/`), aber Tauri 2 packt Sidecars in `Contents/MacOS/` daneben des Hauptbinaries. ADR-0013-Annahme war Tauri-1-Konvention. Bug betraf alle 3 OS-Bundles.

**v2.0.2 Mission:**

**Code-Fixes:**
- `resolve_sidecar` refactored auf `current_exe().parent()`-basierten Lookup, OS-spezifische Pfad-Resolution für macOS .app, Linux AppImage/Deb, Windows .msi/.exe
- Bundle-Integration-Tests pro OS in CI (vorher nur Build-Tests, keine "App startet und findet Sidecars"-Tests — hätte Bug vor Release gefangen)
- Co-Streams-Panel: "Download only"-Action neben "Download & Watch" für UX-Lücke aus Phase 8

**Doku:**
- INSTALL.md macOS-Section: xattr-Befehl klar dokumentiert + Erklärung "App is damaged" (Apple Gatekeeper, kein echter Defekt)
- ADR-0013-Update mit Tauri-2-Sidecar-Layout-Klarstellung (oder ADR-0034 — siehe Session-Report)

**Post-Merge-Incident (PR #23):**
- Test `infra::process::liveness::tests::very_large_pid_is_dead` non-deterministic über procps-ng-Versionen
- `kill -0 4294967295` → truncates to `i32 = -1` (broadcast) auf einem ubuntu-latest Runner-Image, returned success
- Pre-Merge auf PR-Runnern grün, Post-Merge auf main rot
- Test entfernt (Coverage durch `pid_zero_is_never_alive` + `killed_child_reports_dead` redundant)
- PR #23 squash-merged, main wieder grün
- Kein neuer Release-Tag (kein Runtime-Behavior-Change, v2.0.2-Assets unverändert)
